کلاس‌های بوت‌استرپ و عملکرد آن‌ها

  • mb-5: حاشیه پایینی (margin-bottom) به اندازه 3rem (48px) ایجاد می‌کند. برای فاصله زیاد در زیر عنصر استفاده می‌شود.
  • h3: ظاهر تیتر را به سبک تیتر سطح سوم تنظیم می‌کند، حتی اگر تگ دیگری (مثل h2) باشد. فونت کوچکتر از h2.
  • text-primary: رنگ متن را به رنگ اصلی تم (معمولاً آبی #0d6efd) تغییر می‌دهد.
  • h5: ظاهر تیتر را به سبک تیتر سطح پنجم تنظیم می‌کند، حتی اگر تگ دیگری (مثل h3) باشد. فونت کوچکتر از h3.
  • mb-3: حاشیه پایینی به اندازه 1rem (16px) ایجاد می‌کند. برای فاصله متوسط در زیر عنصر.
  • list-group: یک لیست گروهی با استایل مرتب و زیبا ایجاد می‌کند. آیتم‌ها به‌صورت بلوک نمایش داده می‌شوند.
  • list-group-flush: حاشیه‌ها و مرزهای اطراف لیست گروهی را حذف می‌کند برای ظاهری تخت‌تر.
  • card: یک کارت با استایل‌های پایه مانند حاشیه، سایه و گوشه‌های گرد ایجاد می‌کند.
  • bg-light: رنگ پس‌زمینه را به خاکستری روشن (#f8f9fa) تنظیم می‌کند.
  • p-3: پدینگ یکنواخت 1rem (16px) به تمام جهت‌های داخلی عنصر اضافه می‌کند.
  • my-3: حاشیه عمودی (بالا و پایین) به اندازه 1rem (16px) ایجاد می‌کند.
  • h4: ظاهر تیتر را به سبک تیتر سطح چهارم تنظیم می‌کند، حتی اگر تگ دیگری (مثل h3) باشد.
  • card-title: استایل تیتر داخل کارت را تنظیم می‌کند، مناسب برای تیترهای کارت.
  • my-5: حاشیه عمودی (بالا و پایین) به اندازه 3rem (48px) ایجاد می‌کند.
  • btn: یک دکمه با استایل‌های پایه بوت‌استرپ (گوشه‌های گرد، افکت‌ها) ایجاد می‌کند.
  • btn-primary: رنگ دکمه را به رنگ اصلی تم (آبی #0d6efd) تغییر می‌دهد. برای دکمه‌های مهم.
  • mt-3: حاشیه بالایی به اندازه 1rem (16px) ایجاد می‌کند.
  • text-decoration-none: زیرخط پیش‌فرض متن (مثل لینک‌ها) را حذف می‌کند.
  • d-flex: عنصر را به یک کانتینر فلکس‌باکس تبدیل می‌کند برای مدیریت ترازبندی فرزندان.
  • gap-3: فاصله 1rem (16px) بین فرزندان در یک کانتینر فلکس یا گرید ایجاد می‌کند.
مستندات بوت‌استرپ
برای ایجاد فاصله از سمت چپ در یک `div` با استفاده از بوت‌استرپ (Bootstrap)، می‌توانید از کلاس‌های حاشیه (margin) استفاده کنید. به طور خاص، برای فاصله از سمت چپ، از کلاس `ms-*` (در نسخه‌های جدیدتر بوت‌استرپ مثل نسخه ۵) یا `ml-*` (در نسخه‌های قدیمی‌تر مثل نسخه ۴) استفاده می‌شود. ### جزئیات: - **کلاس `ms-*`**: مخفف "margin start" است و در زبان‌های LTR (مثل انگلیسی) به معنای حاشیه از سمت چپ است. مثلاً: - `ms-1` برای حاشیه کوچک - `ms-3` برای حاشیه متوسط - `ms-5` برای حاشیه بزرگ - **کلاس `ml-*`**: در بوت‌استرپ ۴ به معنای "margin left" بود و مشابه `ms-*` عمل می‌کند. - اعداد `*` می‌توانند از ۰ تا ۵ یا `auto` باشند (بسته به مقدار فاصله مورد نظر). ### مثال: ```html
محتوای شما
``` ### نکات: - اگر از بوت‌استرپ ۵ استفاده می‌کنید، `ms-*` را به کار ببرید. - برای بوت‌استرپ ۴، از `ml-*` استفاده کنید. - اگر می‌خواهید فاصله فقط در سایزهای خاصی اعمال شود، از کلاس‌های responsive مثل `ms-md-3` (برای سایز متوسط به بالا) استفاده کنید. اگر نسخه بوت‌استرپ یا جزئیات بیشتری مشخص کنید، می‌توانم دقیق‌تر راهنمایی کنم! برای بررسی نسخه بوت‌استرپ استفاده‌شده در یک وب‌سایت به‌صورت آنلاین و بدون دسترسی به بک‌اند، می‌توانید از ابزارها و روش‌های زیر استفاده کنید. این روش‌ها برای یک فرانت‌اند کار مناسب هستند و نیازی به دسترسی به سرور ندارند: ### ۱. **بررسی فایل CSS بوت‌استرپ در مرورگر** - **مراحل:** 1. وب‌سایت را در مرورگر باز کنید (مثلاً کروم یا فایرفاکس). 2. با کلیک راست و انتخاب **Inspect** یا فشار دادن `Ctrl+Shift+I` (در ویندوز) یا `Cmd+Option+I` (در مک)، ابزار توسعه‌دهندگان (DevTools) را باز کنید. 3. به تب **Network** بروید و گزینه **CSS** را فیلتر کنید. 4. صفحه را رفرش کنید (`Ctrl+R` یا `Cmd+R`) تا فایل‌های CSS لود شوند. 5. به دنبال فایلی با نام‌هایی مثل `bootstrap.min.css` یا `bootstrap.css` بگردید. 6. روی فایل کلیک کنید و محتوای آن را بررسی کنید. معمولاً در خطوط ابتدایی فایل، نسخه بوت‌استرپ در کامنت‌ها ذکر شده است، مثلاً: ```css /*! * Bootstrap v5.3.2 (https://getbootstrap.com/) * Copyright 2011-2023 The Bootstrap Authors */ ``` در این مثال، نسخه بوت‌استرپ ۵.۳.۲ است. - **نکته:** اگر فایل از یک CDN (مثل `cdn.jsdelivr.net` یا `maxcdn.bootstrapcdn.com`) لود شده، URL فایل معمولاً نسخه را مشخص می‌کند، مثلاً: ``` https://stackpath.bootstrapcdn.com/bootstrap/4.5.2/css/bootstrap.min.css ``` اینجا نسخه ۴.۵.۲ است. ### ۲. **استفاده از جاوااسکریپت در کنسول مرورگر** - اگر سایت از نسخه جاوااسکریپت بوت‌استرپ استفاده می‌کند، می‌توانید نسخه را از طریق کنسول بررسی کنید: 1. در DevTools، به تب **Console** بروید. 2. دستور زیر را وارد کنید: ```javascript $.fn.tooltip.Constructor.VERSION ``` یا ```javascript bootstrap.Tooltip.VERSION ``` 3. اگر بوت‌استرپ لود شده باشد، نسخه آن نمایش داده می‌شود (مثلاً `5.3.2`). - **نکته:** این روش برای بوت‌استرپ ۴ و ۵ کار می‌کند، اما در نسخه ۵ ممکن است نیاز به استفاده از `bootstrap` به جای `$` باشد، چون بوت‌استرپ ۵ به jQuery وابسته نیست. ### ۳. **استفاده از ابزارهای آنلاین** - ابزارهایی مثل **Wappalyzer** یا **BuiltWith** می‌توانند فناوری‌های استفاده‌شده در سایت را شناسایی کنند: - **Wappalyzer**: افزونه‌ای برای کروم یا فایرفاکس است. پس از نصب، آیکون آن را در مرورگر کلیک کنید تا فریم‌ورک‌های استفاده‌شده (از جمله بوت‌استرپ) را نشان دهد. گاهی نسخه دقیق را هم مشخص می‌کند. - **BuiltWith**: به وب‌سایت `builtwith.com` بروید، URL سایت را وارد کنید و گزارش فناوری‌ها را بررسی کنید. این ابزار ممکن است نسخه بوت‌استرپ را نشان دهد. - **WhatRuns**: افزونه دیگری است که فریم‌ورک‌ها و نسخه‌های آنها را شناسایی می‌کند. - **محدودیت:** این ابزارها ممکن است نسخه دقیق را همیشه نشان ندهند، به‌خصوص اگر بوت‌استرپ به‌صورت محلی (نه از CDN) استفاده شده باشد. ### ۴. **بررسی نشانه‌های خاص نسخه در DOM یا CSS** - هر نسخه بوت‌استرپ کلاس‌ها و ویژگی‌های خاصی دارد که می‌توانید با بررسی DOM یا استایل‌ها آن‌ها را شناسایی کنید: - **بوت‌استرپ ۳**: استفاده از کلاس‌هایی مثل `col-xs-*`، `navbar-default`، یا `panel`. - **بوت‌استرپ ۴**: استفاده از کلاس‌هایی مثل `col-*` (بدون `xs`)، `navbar-expand-*`، یا `card` به جای `panel`. - **بوت‌استرپ ۵**: استفاده از کلاس‌هایی مثل `ms-*` (به جای `ml-*`)، حذف کامل jQuery، و پشتیبانی از `rtl` به‌صورت داخلی. - در DevTools، به تب **Elements** بروید و کلاس‌های استفاده‌شده در HTML را بررسی کنید. برای مثال، وجود `ms-3` نشان‌دهنده بوت‌استرپ ۵ است. ### ۵. **استفاده از وب‌سایت رسمی بوت‌استرپ یا مستندات** - اگر فایل CSS یا JS را پیدا کردید، می‌توانید نام فایل یا بخشی از کد را در مستندات رسمی بوت‌استرپ (`getbootstrap.com`) جستجو کنید تا نسخه را شناسایی کنید. - همچنین، اگر سایت از CDN استفاده می‌کند، نسخه معمولاً در URL فایل مشخص است (مثل `bootstrap/5.3.2/`). ### نکات اضافی: - **اگر بوت‌استرپ شخصی‌سازی شده باشد:** ممکن است نسخه در فایل CSS ذکر نشود. در این حالت، بررسی کلاس‌های خاص یا استفاده از ابزارهای آنلاین بهترین گزینه است. - **بررسی RTL:** اگر سایت فارسی است و از بوت‌استرپ راست‌چین‌شده استفاده می‌کند، فایل CSS ممکن است شامل `rtl` باشد (مثل `bootstrap.rtl.min.css`). این بیشتر در بوت‌استرپ ۵ رایج است. - **محدودیت دسترسی:** چون شما فقط به فرانت‌اند دسترسی دارید، روش‌های بالا معمولاً کافی هستند. اگر هیچ‌کدام نتیجه نداد، می‌توانید از تیم بک‌اند بخواهید فایل‌های پروژه (مثل `package.json` یا پوشه `node_modules`) را بررسی کنند. ### ابزار پیشنهادی: برای سریع‌ترین نتیجه، پیشنهاد می‌کنم از **Wappalyzer** یا بررسی فایل CSS در DevTools استفاده کنید. این روش‌ها معمولاً دقیق و سریع هستند. اگر سوال یا نیاز به توضیح بیشتری دارید، بگویید تا دقیق‌تر راهنمایی کنم![](https://replywp.com/bootstrap-installation-5/)
